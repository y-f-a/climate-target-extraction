# TASKS

- Canonical task board: `docs/TASKS.md`
- Status values: `todo`, `in_progress`, `blocked`, `done`
- Required fields per task: `id`, `title`, `status`, `owner`, `target_date`, `evidence_link`, `notes`
- `target_date` is optional for all statuses
- `evidence_link` is required for `done` and `blocked`; optional otherwise
- Owner must be a single primary owner (`user`, `codex`, or named person)
- Tasks move manually between sections; when status changes, add a short note
- ID format is sequential: `T-001`, `T-002`, ...
- Current board sequence reset for strategy-aligned execution starts at `T-001`
- Evidence for experiment tasks should be artifact paths (manifest/report/comparison outputs), not notes-only
- Manual-first tracking templates are versioned in `docs/` (not `artifacts/`)
